# 文字コード（字符编码）

---

## 🌿基本概念
 
计算机通过预先定义的 **映射规则**，将二进制解释为对应的字符，从而实现文字的显示与处理。
 
这个规则就被称为： 📘 **字符编码（文字コード）**

---

## 🌿 常见的字符编码

### ASCII编码

>ASCII 是最早的字符编码标准之一：  
仅支持约 128 个字符，仅能表示英文字符，  
无法覆盖多语言环境，因此在实际应用中不够使用。

---

### Shift_JIS

>Shift_JIS 是一种主要在日本旧 Windows 环境中使用的字符编码。  
其支持日文字符并与 ASCII 英文部分兼容  
但在不同环境,非日本系统中兼容性较差, 因此逐渐被淘汰  


---

### UTF-8

>UTF-8 是目前全球通用的字符编码标准.  
支持几乎所有语言,与 ASCII 完全兼容, 广泛应用于：Web开发 ,操作系统, 以及各类编程环境

---

## 🌿 统一UTF-8编码形式
编译器,控制台,文件有一个不统一编码,都有可能引起乱码

### 🍎 项目编译器设置
- 在项目属性中添加命令,使编译器按照指定的编码去“读取源码文件”
- 编译器设置决定代码解析方式, 不会改变文件本身的编码  

项目 → 属性 → c/c++ → 命令行
![项目编译器设置命令](/PG3/00_01_文字コード/image5.png)

---

### 🍎 控制台编码设置
- 在程序中调用函数，使控制台按照指定编码显示输出内容
- 不影响文件编码或编译过程,仅作用于程序运行时的**控制台窗口**

```cpp
#include <Windows.h>
int main() {
    SetConsoleOutputCP(65001);
    return 0;
}
```
---

### 🍎 文件编码设置 (保存为UTF-8)

- 通过“使用编码保存”可以指定文件的实际存储编码  
- 文件编码决定源码在磁盘中的存储方式  

文件 → 另存为 → 使用编码保存
![Visual Studio保存页面](/PG3/00_01_文字コード/image2.png)

常见 UTF-8 选项有两个：

- Unicode (UTF-8 无签名)  
![Visual Studio保存页面](/PG3/00_01_文字コード/image3.png)

- Unicode (UTF-8 带签名)  
![Visual Studio保存页面](/PG3/00_01_文字コード/image4.png)

    >签名（BOM）是文件开头的一段特殊标记，  
    用于说明文件所使用的字符编码格式。  
    在 UTF-8（带 BOM）中，文件开头会包含额外的 3 个字节：EF BB BF

    >在某些微软软件（如 Excel）中，如果没有 BOM 可能会出现乱码  
    BOM 也可以用于标识文件编码格式  

虽然带签名可以帮助某些软件识别编码（如 Excel），  
但在编程环境中可能会导致解析问题或兼容性问题。  

因此推荐使用: **Unicode (UTF-8 无签名)**

>KamataEngine（Novice）也默认使用 UTF-8（无 BOM）

.editorconfig 是用于统一代码编辑规则的配置文件。  

---

### 🧠 作用

在开发过程中，不同开发者的编辑器设置可能不同，  
容易导致文件编码不一致，从而引发乱码问题。  

使用 .editorconfig 可以做到：

> 项目中的所有文件在保存时自动按统一规则处理  

---

### 🍎 创建 .editorconfig

#### 操作步骤

1. 右键解决方案  
2. 选择「添加」→「新建项」  
3. 选择文本文件  
4. 命名为 `.editorconfig`  

---

### 🍎 编写配置内容

```
[*.{cpp,h}]
charset = utf-8
```

---

### 🧠 说明

- `[*.{cpp,h}]` 表示匹配所有 `.cpp` 和 `.h` 文件  
- `charset = utf-8` 表示统一使用 UTF-8 编码  

---

### ⚠️ 注意

- 配置在**保存文件时生效**  
- 可能会自动修改已有文件的编码  
- 应在项目初期统一设置，避免后期混乱  

---

### 🍎 使用效果

在项目中添加 .editorconfig 后：

- 新文件默认使用 UTF-8  
- 保存文件时自动统一编码  
- 避免不同文件编码不一致  

---

### 🎯 总结

.editorconfig 用于统一项目中的编码和编辑规则，  
可以在保存文件时自动应用设置，  
是团队开发中保证编码一致性的有效工具。

## 🌿 关于 Unicode 与 UTF-8

- Unicode：为每个字符分配唯一编号（字符标准）  
- UTF-8：将这些编号转换为字节进行存储（编码方式）  

        字符 → Unicode编号 → UTF-8编码
        あ → U+3042 → E3 81 82

UTF-8 是最常用的编码方式, 还有 UTF-16、UTF-32 等其他编码
