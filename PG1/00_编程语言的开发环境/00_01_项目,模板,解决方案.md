## 项目模板

在创建项目时，需要选择一个“项目模板”。

![](/PG1/00_编程语言的开发环境/image1.png)

项目模板可以理解为已经帮你准备好的一套“项目结构 + 配置”

Visual Studio 提供多种语言、平台和用途的模板.

---

### C++ 空项目

空项目可以理解为一张“没有任何配置的白纸”。

在创建空项目时，Visual Studio 并不会为你决定程序的运行类型，
也不会生成入口函数，需要开发者自行完成这些设置。

>在 Windows 程序中，程序的运行类型由“子系统（Subsystem）”决定：
>- Console（控制台程序）
>- Windows（图形界面程序）  
>>不同子系统对应不同的入口函数：
>>- Console → 使用 `main()`  
>>- Windows → 使用 `WinMain()`  

---

### C++ 控制台应用

可以理解为已经帮你配置好“控制台子系统”的空项目

- 自动包含库 `iostream` 

    >iostream 是 C++ 标准库中的输入输出头文件，
    >用于实现基于“流（stream）”的输入和输出操作。
    >与 C 语言中的 stdio.h 不同，
    >iostream 使用面向对象的方式（如 cout 和 cin）进行输入输出。

- 自动使用 `main()`作为入口

    ```cpp
    int main() {
        return 0;
    }
    ```

---

### C++ Windows 桌面应用程序

可以理解为已经帮你配置好“Windows 子系统”的项目  

- 自动包含库 `windows.h`  

  >windows.h 是 Windows 提供的系统库  
  >用于创建窗口、处理输入、控制程序运行等  

- 自动使用 `WinMain()` 作为入口  

    ```cpp
    #include <windows.h>

    int WINAPI WinMain(HINSTANCE hInstance, HINSTANCE, LPSTR, int nCmdShow) {
        MessageBox(NULL, "Hello Windows", "提示", MB_OK);
        return 0;
    }
    ```

- 初学使用的 Kamata（Novice）引擎,也是建立在“Windows子系统（WinMain）”上的程序

## 项目和解决方案

### 项目（Project）

项目是实际存放代码和配置的单位，可以理解为一个工程文件夹。

![](/PG1/00_编程语言的开发环境/image2.png)

一个项目通常包含：

- 源文件（.cpp）
- 头文件（.h）
- 项目配置文件（.vcxproj 等）
- 编译生成的程序（.exe）

---

### 解决方案（Solution）

解决方案（Solution）表示完成一个程序或目标所需的整体结构，  
通常由一个或多个项目（Project）组成。

例如：

```text
做游戏 = 解决“如何实现游戏逻辑”

解决方案（Solution）
├── 玩家模块（Project）
        └──玩家.cpp
        └──玩家.h
├── 敌人模块（Project）
├── 场景模块（Project）
```

初学阶段一个解决方案通常只包含一个项目

`.sln` 是 Solution 的缩写，用于表示解决方案文件，
它负责记录项目结构、路径以及启动信息等内容。

无论是单个项目还是多个项目，只要它们共同服务于同一个开发目标，
都可以被统一归纳为一个解决方案。

在Visual Studio有名为解决方案资源管理器的窗口,可以让你很清楚的看到整个解决方案的结构和进行各种操作
